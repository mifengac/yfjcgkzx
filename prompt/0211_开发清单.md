#### 2.1 `count_jq_by_diqu` 函数（第88-130行）
**修改内容：**
- 将 `FROM "ywdata"."v_jq_optimized" vjo` 替换为带 CTE 的查询
- 使用 CTE `flat_config` 展开 `case_type_config` 表的 `newcharasubclass_list`
- 通过 JOIN 关联 `zq_kshddpt_dsjfx_jq` 和 `flat_config`
- 将 `vjo."leixing"` 改为 `fc."leixing"`
- 将 `vjo."cmdid"` 改为 `jq."cmdid"`
- 将 `vjo."calltime"` 改为 `jq."calltime"`
- 使用 `COUNT(DISTINCT (jq.id, fc.code_type))` 进行去重统计

**关键SQL结构：**
```sql
WITH flat_config AS (
    SELECT ctc.leixing,
        unnest(ctc.newcharasubclass_list) AS single_code,
        'original' AS code_type
    FROM ywdata.case_type_config ctc
    UNION ALL
    SELECT ctc.leixing,
        unnest(ctc.newcharasubclass_list) AS single_code,
        'confirmed' AS code_type
    FROM ywdata.case_type_config ctc
)
SELECT LEFT(jq."cmdid", 6) AS diqu, COUNT(DISTINCT (jq.id, fc.code_type)) AS cnt
FROM "ywdata"."zq_kshddpt_dsjfx_jq" jq
JOIN flat_config fc ON (
    (jq.neworicharasubclass = fc.single_code AND fc.code_type = 'original')
    OR (jq.newcharasubclass = fc.single_code AND fc.code_type = 'confirmed')
)
WHERE jq."calltime" BETWEEN %s AND %s
```

### 3. 技术要点

#### 3.1 数据模型理解
- **原视图逻辑**：`v_jq_optimized` = `zq_kshddpt_dsjfx_jq` + `case_type_config` JOIN
- **leixing 字段来源**：通过 JOIN `case_type_config` 获得
- **去重策略**：一条警情记录可能匹配多个配置项，需按 (id, code_type) 去重

- ✅ 类型过滤功能正常（通过 `fc."leixing" = ANY(%s)`）

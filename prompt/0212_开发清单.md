# 开发清单：hqzcsj 模块“未成年人统计”数据源调整（0212）

## 1. 目标与范围
- 目标页面：`hqzcsj` 模块 Tab 页 `未成年人统计`。
- 目标文件：
  - `hqzcsj/dao/zfba_wcnr_jqaj_dao.py`
  - `hqzcsj/service/zfba_wcnr_jqaj_service.py`
  - `hqzcsj/routes/zfba_wcnr_jqaj_routes.py`
  - `hqzcsj/templates/zfba_wcnr_jqaj_tab.html`
- 复用来源：`矫治情况统计` 当前口径（`hqzcsj/dao/jzqk_tongji_dao.py` 的 `fetch_jzqk_data` 输出字段语义）。

## 2. 指标口径调整清单

### 2.1 行政嫌疑人 / 刑事嫌疑人
- 新数据源：`ywdata.v_wcnr_wfry_base`。
- 统计规则：
  - 行政嫌疑人：`案件类型='行政'`。
  - 刑事嫌疑人：`案件类型='刑事'`。
- 分组规则：按 `地区` 列分组（示例：`445302`、`445303`）。
- 类型过滤：按 `案由` 字段匹配 `case_type_config.ay_pattern`（与现有 leixing 逻辑保持一致）。
- 同比：`同比行政嫌疑人`、`同比刑事嫌疑人` 同口径切换。

### 2.2 训诫书 -> 矫治文书
- 字段改名：
  - 展示列：`训诫书` 改为 `矫治文书`。
  - 展示列：`同比训诫书` 改为 `同比矫治文书`。
- 新数据源：复用“矫治情况统计”口径数据（同一明细集）。
- 统计规则：`是否开具矫治文书='是'`。
- 分组规则：按 `地区` 列分组。
- 类型过滤：按 `案由` 字段过滤。
- 同比：`同比矫治文书` 同口径切换。

### 2.3 加强监督教育
- 新数据源：复用“矫治情况统计”口径数据。
- 统计规则：`是否开具家庭教育指导书='是'`。
- 分组规则：按 `地区` 列分组。
- 类型过滤：按 `案由` 字段过滤。
- 同比：`同比加强监督教育` 同口径切换。

### 2.4 符合送校 / 送校
- 新数据源：复用“矫治情况统计”口径数据。
- 统计规则：
  - 符合送校：`是否符合送生='是'`。
  - 送校：`是否送校='是'`。
- 分组规则：按 `地区` 列分组。
- 类型过滤：按 `案由` 字段过滤。
- 同比要求：
  - `送校`、`同比送校` 同口径切换。
  - `符合送校` 无同比列，保持现状。

## 3. 代码改造任务（按文件）

### 3.1 `hqzcsj/dao/zfba_wcnr_jqaj_dao.py`
- 新增可复用查询函数：基于 `v_wcnr_wfry_base` + 现有类型过滤口径统计以下指标。
  - `count_wfry_by_diqu_and_ajlx(...)`：行政/刑事嫌疑人（当前+同比共用）。
  - `count_jzqk_flag_by_diqu(...)`：按 flag 字段统计（`是否开具矫治文书`、`是否开具家庭教育指导书`、`是否符合送生`、`是否送校`）。
- 替换或废弃旧函数调用来源：
  - `count_wcnr_xyr_by_diqu_and_ajlx`
  - `count_wcnr_xjs_by_diqu`
  - `count_wcnr_jtjyzdtzs_by_diqu`
  - `count_fuhe_songxiao_by_diqu`
  - `count_songxiao_by_diqu`
- 明细查询 `fetch_detail_rows(...)` 调整：
  - 指标 `行政嫌疑人`、`刑事嫌疑人` 改走 `v_wcnr_wfry_base` 明细。
  - 指标 `矫治文书`、`加强监督教育`、`符合送校`、`送校` 改走同一明细源（按对应 flag='是' 过滤）。
  - 兼容旧 metric：保留 `训诫书` 到 `矫治文书` 的别名映射（避免历史链接/导出调用报错）。

### 3.2 `hqzcsj/service/zfba_wcnr_jqaj_service.py`
- `build_summary(...)` 指标取数改造：
  - 行政/刑事嫌疑人：调用新 `v_wcnr_wfry_base` 统计函数。
  - 矫治文书 / 加强监督教育 / 符合送校 / 送校：调用统一 flag 统计函数。
- 返回字段改名：
  - `训诫书` -> `矫治文书`
  - `同比训诫书` -> `同比矫治文书`
- 同比逻辑保持：除 `符合送校` 外，其余相关同比字段全部切换到新口径。

### 3.3 `hqzcsj/templates/zfba_wcnr_jqaj_tab.html`
- 表头列改名：
  - `训诫书` -> `矫治文书`
  - `同比训诫书` -> `同比矫治文书`
- `METRIC_BY_COL` 映射更新：前端点击明细跳转 metric 改为新值。
- 确认“同比列点击”仍能正确落到同比时间段。

### 3.4 `hqzcsj/routes/zfba_wcnr_jqaj_routes.py`
- `export_detail_all` 的 `metrics` 列表同步改名：
  - `训诫书` -> `矫治文书`
- 若保留别名兼容，确保导出全量仍可正确生成对应 sheet。

## 4. 验证与测试清单

### 4.1 功能验证
- 无类型筛选：
  - 各地区 + 全市合计正常。
  - 新口径指标值可返回。
- 有类型筛选（单选/多选）：
  - 指标受 `案由` 过滤影响。
  - 地区统计与全市合计一致。
- 明细跳转：
  - 当前值点击进入当前时间段明细。
  - 同比值点击进入同比时间段明细。

### 4.2 口径验收
- 抽样核对 3 个地区（建议：`445302`、`445303`、`445381`）：
  - 行政嫌疑人/刑事嫌疑人是否严格等于 `v_wcnr_wfry_base` 分组计数。
  - 矫治文书/加强监督教育/符合送校/送校是否严格按 flag='是'。
- 确认“符合送校无同比列”维持不变。

### 4.3 导出验证
- 汇总导出（xlsx/csv）列名正确，`矫治文书` 生效。
- 明细导出（单指标、全部）无报错，sheet 名与指标一致。

## 5. 风险点
- 现有 `v_wcnr_wfry_base` 中 `地区` 是否始终为 6 位代码，若存在名称或空值需统一清洗。
- `案由` 与 `ay_pattern` 的匹配精度（SIMILAR TO）需抽样验证，避免遗漏或误匹配。
- 旧 metric `训诫书` 若外部仍在调用，未做兼容会导致 detail/export 参数失效。

## 6. 建议的实施顺序
1. 先改 DAO（统一新口径函数 + 明细）。
2. 再改 Service（汇总字段/同比映射）。
3. 再改前端模板与路由 metric 列表（字段改名与导出）。
4. 最后做联调验收（页面点击 + 导出 + 抽样核对）。

## 7. 完成定义（DoD）
- 页面“未成年人统计”中以下指标全部按新口径返回：
  - 行政嫌疑人、刑事嫌疑人
  - 矫治文书（原训诫书）
  - 加强监督教育
  - 符合送校
  - 送校
- 除“符合送校”外，相关同比列均按相同新口径计算。
- 汇总/明细/导出三条链路均可用，且字段名称与需求一致。

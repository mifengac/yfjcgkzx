# 开发清单：导出「飙车炸街」日报（jingqing_anjian）

## 目标
在 `jingqing_anjian` 模块的“案件查询与分析”Tab 内，新增“导出飙车炸街”按钮：用户上传 `xlsx` 附件后，后端解析并清洗指定列数据，使用 `docxtpl` 将数据写入 `jingqing_anjian/templates/biaochezhajie_ribao.docx`，最后将生成的 `docx` 文件下载到浏览器。

## 需求要点（来自 prompt/1227.md）
1. 新增按钮位置：在“导出警情研判”按钮前面新增“导出飙车炸街”按钮。
2. 点击按钮后弹出文件上传框。
3. 仅允许上传 `xlsx`：
   - 若不是 `xlsx` 直接提示并停止。
4. 解析上传的 `xlsx` 得到 `result`（按单元格坐标读取）。
5. 清洗列：`C,K,L,N,R,U,V`（同一套规则）：
   - `>0`：在原值前加“上升”（如 `2.22%` → `上升2.22%`）
   - `<0`：在原值前加“下降”，并取绝对值（如 `-11.11%` → `下降11.11%`）
   - `=0`：写为“持平”（如 `0.00%` → `持平`）
6. `docxtpl` 模板：`jingqing_anjian/templates/biaochezhajie_ribao.docx`
   - `{{time}}`：当日日期（例：`2025年12月27日`）
   - `{{last_time}}`：前一日日期
   - `{{this_week}}`：本周一至昨天（例：`2025年12月22日-12月26日`）
   - `{{B7}}`、`{{C7}}` 等：对应单元格值
   - `{result1}...{result6}`：二次拼接字符串（先写到 `temp.md`，你稍后填入 docx）

## 开发拆解（前后端）
### 前端（模板 + JS）
1. 修改 `jingqing_anjian/templates/jingqing_anjian.html`
   - 在“导出警情研判”按钮前插入“导出飙车炸街”按钮。
   - 触发隐藏的 `<input type="file" accept=".xlsx">` 选择文件。
2. 发送上传请求
   - 使用 `fetch`/`axios`（按现有项目习惯）提交 `multipart/form-data`。
   - 上传过程中显示遮罩/禁用按钮，失败提示原因。
3. 下载文件
   - 后端返回 `docx`，前端使用 blob 下载（文件名规则待确认）。

### 后端（路由 + service）
1. 新增接口（建议）
   - `POST /jingqing_anjian/api/biaochezhajie/export`
   - 入参：表单文件 `file`
   - 出参：`application/vnd.openxmlformats-officedocument.wordprocessingml.document`
2. 文件校验
   - 校验扩展名/`mimetype`（双重校验，错误返回友好信息）。
3. 解析 xlsx
   - 使用 `openpyxl` 读取第一个 sheet（或按你确认的 sheet）。
   - 读取模板所需的单元格（如 `B7` 这类坐标）与范围（如 `A8:C12` 等）。
4. 数据清洗
   - 对列 `C,K,L,N,R,U,V` 按规则清洗（注意：原值可能是数字/百分比字符串/空值）。
5. 组装模板上下文
   - 基础变量：`time/last_time/this_week` + 所有单元格变量（`B7` 等）
   - 二次拼接：`result1..result6`（规则记录在 `temp.md`）
6. 渲染并下载
   - `docxtpl` 渲染 `biaochezhajie_ribao.docx`
   - 以临时文件/内存流返回下载（避免落盘或落盘后及时清理）

## 交付物
1. 前端：新增“导出飙车炸街”按钮与上传/下载交互。
2. 后端：新增导出接口 + xlsx 解析/清洗 + docxtpl 渲染下载。
3. 文档：`jingqing_anjian/templates/temp.md`（二次拼接变量规则草稿）。

## 自测清单（不涉及内网）
1. 上传非 xlsx：前端提示 + 后端返回 4xx（信息明确）。
2. 上传示例 `biaochezhajieribao_fujian.xlsx`：成功下载 docx。
3. 校验清洗：C/K/L/N/R/U/V 列出现 “上升/下降/持平” 是否符合预期。
4. 校验时间变量：`time/last_time/this_week` 格式正确。

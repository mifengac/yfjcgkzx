# 开发清单：未成年人警情案件统计新增“场所案件/同比场所案件”（0212增量）

## 1. 目标
- 模块：`hqzcsj` → `未成年人警情案件统计` tab。
- 在 `同比刑事` 后新增两列：`场所案件`、`同比场所案件`。
- 统计口径：
  - 基础查询与 `行政/刑事` 同源（`zq_zfba_wcnr_ajxx`）
  - 但**不按 `ajxx_ajlx` 过滤**
  - 对查询结果按地址做模型分类，仅保留 `分类结果='重点管控行业'` 后计数。

## 2. 口径定义
### 2.1 场所案件（当前）
1. 时间：当前时间窗（`start_time ~ end_time`）。
2. 类型过滤：沿用现有 `案由` 模式过滤（`ay_pattern`）。
3. 业务过滤：沿用 `行政/刑事` 现有约束（建议保持一致）：
   - `ajxx_ajzt NOT IN ('已撤销','已合并')`
   - `ajxx_cbdw_mc !~ '交通'`
4. 地址分类：使用 `xunfang/5lei_dizhi_model`（通过 `xunfang.service.jiemiansanlei_service.predict_addresses`）。
5. 保留规则：`分类结果='重点管控行业'`。
6. 统计：按地区（`LEFT(ajxx_cbdw_bh_dm, 6)`）计数。

### 2.2 同比场所案件
- 时间改为同比窗口（`yoy_start_time ~ yoy_end_time`），其余口径完全一致。

## 3. SQL 基础查询（示意）
```sql
SELECT
  ajxx_ajbh AS "案件编号",
  ajxx_jqbh AS "警情编号",
  ajxx_ajmc AS "案件名称",
  ajxx_ajlx AS "案件类型",
  ajxx_aymc AS "案由名称",
  ajxx_lasj AS "立案时间",
  ajxx_cbdw_mc AS "承办单位",
  LEFT(ajxx_cbdw_bh_dm, 6) AS "地区",
  ajxx_fadd AS "发案地点"
FROM ywdata.zq_zfba_wcnr_ajxx
WHERE ajxx_lasj BETWEEN %s AND %s
  AND ajxx_ajzt NOT IN ('已撤销','已合并')
  AND ajxx_cbdw_mc !~ '交通'
  AND 1=1
  -- + 案由模式过滤（按 leixing -> ay_pattern）
```
说明：与 `行政/刑事` 的核心差异是**不加 `ajxx_ajlx = '行政/刑事'` 条件**。

## 5. 性能与稳定性建议（建议采用）
1. 批量分类：一次性对当前窗口全量地址分类，再按地区聚合；避免“按地区循环+重复推理”。
2. 地址去重缓存：对 `发案地点` 去重后推理，再回填，减少模型推理次数。
3. 模型加载失败兜底：明确抛错信息（沿用现有“无法加载地址分类模型”）。
4. 空地址处理：空字符串地址不计入“重点管控行业”。

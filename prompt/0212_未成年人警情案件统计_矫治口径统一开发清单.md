# 开发清单：未成年人警情案件统计（矫治口径统一改造）

## 1. 目标范围
- 页面：`hqzcsj` 模块 `未成年人警情案件统计` tab。
- 核心目标：将 `矫治文书` 及其后续相关指标统一切换到 `矫治情况统计` 同口径数据源（`jzqk_tongji` 当前 SQL 逻辑）。
- 追加目标：新增行政/刑事嫌疑人拆分指标，并将矫治/监督教育按行政与刑事拆分展示。

## 2. 指标口径映射（目标口径）
### 2.1 新增指标
- `嫌疑人(行政)`、`同比嫌疑人(行政)`
  - 来源：`ywdata.v_wcnr_wfry_base` 同口径（即 `jzqk_tongji` 明细结果）
  - 条件：`案件类型='行政'`
- `嫌疑人(刑事)`、`同比嫌疑人(刑事)`
  - 来源：同上
  - 条件：`案件类型='刑事'`

### 2.2 指标拆分改名
- 原：`矫治文书`、`同比矫治文书`
- 新：
  - `矫治文书(行政)`、`同比矫治文书(行政)`
    - 条件：`案件类型='行政'` 且 `是否开具矫治文书='是'`
  - `矫治文书(刑事)`、`同比矫治文书(刑事)`
    - 条件：`案件类型='刑事'` 且 `是否开具矫治文书='是'`

- 原：`加强监督教育`、`同比加强监督教育`
- 新：
  - `加强监督教育(行政)`、`同比加强监督教育(行政)`
    - 条件：`案件类型='行政'` 且 `是否开具家庭教育指导书='是'`
  - `加强监督教育(刑事)`、`同比加强监督教育(刑事)`
    - 条件：`案件类型='刑事'` 且 `是否开具家庭教育指导书='是'`

### 2.3 送校相关
- `符合送校`：改为按 `是否符合送生='是'` 统计（沿用 `jzqk_tongji` 最新规则）
- `送校`、`同比送校`：按 `是否送校='是'` 统计

## 3. 列结构调整建议（前端展示）
- 在 `同比刑事` 后新增：
  - `嫌疑人(行政)`、`同比嫌疑人(行政)`
  - `嫌疑人(刑事)`、`同比嫌疑人(刑事)`
- 将原有列替换为拆分列：
  - `矫治文书/同比矫治文书` -> 4列（行政/刑事 + 同比）
  - `加强监督教育/同比加强监督教育` -> 4列（行政/刑事 + 同比）
- `符合送校`、`送校`、`同比送校` 保留现有位置（仅换口径）。

## 4. 代码改造清单（按文件）

- 更新 `METRIC_BY_COL`：
  - 新增 `嫌疑人(行政/刑事)` 映射。
  - 新增拆分后的 `矫治文书(行政/刑事)`、`加强监督教育(行政/刑事)` 映射。

  - `嫌疑人(行政/刑事)`
  - `矫治文书(行政/刑事)`
  - `加强监督教育(行政/刑事)`
  - `符合送校`（`是否符合送生='是'`）

### 4.3 `hqzcsj/dao/zfba_wcnr_jqaj_dao.py`

- 确认导出 sheet 名长度与可读性。

## 5. 验收清单
1. 页面列名与顺序符合需求，新增/替换列正确显示。
2. `嫌疑人(行政/刑事)` 与 `v_wcnr_wfry_base` 同口径按案件类型过滤结果一致。
3. `矫治文书(行政/刑事)` 与 `是否开具矫治文书='是'` + 案件类型过滤一致。
4. `加强监督教育(行政/刑事)` 与 `是否开具家庭教育指导书='是'` + 案件类型过滤一致。
5. `符合送校` 与 `是否符合送生='是'` 一致（包含你刚修改后的刑事/行政判定逻辑）。
6. `送校/同比送校` 与 `是否送校='是'` 一致。
7. 明细点击与导出（单项/全量）全部可用、无报错。

## 6. 风险与建议
- 风险：旧前端缓存或外部调用仍传旧 metric 名，可能导致明细/导出失败。
- 建议：
  - 后端维持 1 个迭代周期的旧 metric 兼容映射。
  - 将“按案件类型 + flag 字段聚合”的逻辑抽成统一 helper，减少未来字段扩展改动量。

# 开发清单：hqzcsj「未成年人警情案件统计」改造（0212）

## 1. 目标
- 将 `hqzcsj` 模块 Tab 名称从“未成年人统计”调整为“未成年人警情案件统计”。
- 在未成年人统计表中新增 `转案数`、`同比转案数`。
- 删除 `行政嫌疑人`、`同比行政嫌疑人`、`刑事嫌疑人`、`同比刑事嫌疑人` 四列及其对应数据链路。

## 2. 口径定义
### 2.1 转案数（当前）
- 参考 `警情案件统计` 的“转案”口径。
- 统计方式：在“警情”基础数据中，按警情编号去重后，仅统计已关联案件编号的警情。
- 案件编号非空判断：
  - `NULLIF(BTRIM(zza."ajxx_ajbh"), '') IS NOT NULL`

### 2.2 同比转案数
- 时间窗口按现有同比窗口规则（与 `同比警情` 一致）。
- 统计口径与 `转案数` 完全一致，仅时间区间不同。

## 3. 改造清单（按文件）
### 3.1 页面主 Tab 文案
- 文件：`hqzcsj/templates/hqzcsj_zongcha.html`
- 修改：
  - `data-tab="wcnr"` 对应按钮文案：`未成年人统计` -> `未成年人警情案件统计`

### 3.2 未成年人统计页列结构
- 文件：`hqzcsj/templates/zfba_wcnr_jqaj_tab.html`
- 修改：
  - 新增列：`转案数`、`同比转案数`
  - 删除列：`行政嫌疑人`、`同比行政嫌疑人`、`刑事嫌疑人`、`同比刑事嫌疑人`
  - 更新列到指标映射（明细点击 `METRIC_BY_COL`）与渲染顺序，确保导出和跳转一致。

### 3.3 汇总服务层
- 文件：`hqzcsj/service/zfba_wcnr_jqaj_service.py`
- 修改：
  - 汇总返回字段移除：`行政嫌疑人`、`同比行政嫌疑人`、`刑事嫌疑人`、`同比刑事嫌疑人`
  - 汇总返回字段新增：`转案数`、`同比转案数`
  - 全市与各地区行都按新字段组装。

### 3.4 DAO 数据层
- 文件：`hqzcsj/dao/zfba_wcnr_jqaj_dao.py`
- 修改：
  - 新增或复用“按地区统计转案数”函数（当前/同比共用，传入不同时间区间）。
  - SQL 关键点：
    - 警情源：沿用未成年人页当前警情口径（与页面现有过滤一致）
    - 关联案件：`LEFT JOIN ywdata.zq_zfba_ajxx zza ON jq."caseno" = zza."ajxx_jqbh"`
    - 转案条件：`NULLIF(BTRIM(zza."ajxx_ajbh"), '') IS NOT NULL`
    - 去重：按警情编号去重计数
  - 删除或停用未成年人页中“行政/刑事嫌疑人”相关计数函数调用。

### 3.5 路由与导出
- 文件：`hqzcsj/routes/zfba_wcnr_jqaj_routes.py`
- 修改：
  - 汇总导出文件名：`未成年人统计{ts}` -> `未成年人警情案件统计{ts}`
  - `export_detail_all` 指标列表同步：
    - 删除：`行政嫌疑人`、`刑事嫌疑人`
    - 新增：`转案数`

## 4. 验收清单
1. 页面 Tab 文案显示“未成年人警情案件统计”。
2. 表格列中新增 `转案数/同比转案数`，且不再出现 4 个嫌疑人列。
3. `转案数` 抽样核对：与同时间范围下“警情去重后且案件编号非空”的结果一致。
4. `同比转案数` 抽样核对：与同比时间窗口按同口径一致。
5. 汇总导出和明细导出无报错，列头与指标一致。

## 5. 风险与建议
- 风险：`ajxx_ajbh` 可能存在空格串，必须使用 `NULLIF(BTRIM(...), '') IS NOT NULL`，避免漏统。
- 风险：若前端/外部脚本仍依赖被删除的4个字段，可能出现空列或解析失败。
- 建议（兼容性）：
  - 短期可在后端保留旧字段但固定返回 `0`（仅接口兼容，不在前端展示），观察一周后再彻底移除。
  - 将“转案口径SQL片段”抽成公共函数，供 `zfba_jq_aj_dao.py` 与 `zfba_wcnr_jqaj_dao.py` 复用，减少后续口径漂移。

## 6. 实施顺序
1. 先改 DAO（新增转案统计、移除嫌疑人取数调用）。
2. 再改 Service（汇总字段替换）。
3. 再改模板与路由（列、导出、文案）。
4. 最后做联调验收（页面、明细跳转、导出）。
